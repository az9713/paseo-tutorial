  вҳ… Insight в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  - Your iPhone never talks to Claude Code directly. It connects to the Paseo
  daemon on this PC; the daemon spawns and drives the claude CLI you already
  have. The phone is just a remote control for agents that run here.
  - Pairing is relay-based, not LAN-based. The QR encodes a connection through
  Paseo's relay (wss://relay.paseo.sh:443) plus your daemon's public key. So it
  works from anywhere вҖ” cellular, a different WiFi вҖ” not just your home network.
  Traffic is end-to-end encrypted; the relay is zero-knowledge and can't read
  your code or prompts (docs/architecture.md, SECURITY.md).
  - That's why the daemon status earlier showed a Relay row вҖ” relay is on by
  default, which is what makes phone pairing possible.
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  The flow

  1. Get the Paseo app on your iPhone
  - App Store вҶ’ search "Paseo", or
  - Open https://app.paseo.sh in Safari вҶ’ Share вҶ’ Add to Home Screen (the PWA
  works the same way).

  2. On this PC, start the daemon and show the pairing QR. Run this in your own
  git bash terminal (so the QR renders sharply enough to scan):

  paseo onboard

  This is the all-in-one first-run command вҖ” it starts the daemon, waits until
  it's ready, then prints a QR code + a pairing link. (It'll ask once whether to
  enable voice features; answer No if you don't want the local speech-model
  download.)

  If the daemon is already running and you just want the code again later:

  paseo daemon pair

  3. Scan the QR with the Paseo iOS app (or paste the pairing link if scanning
  is awkward). The phone connects to your daemon over the encrypted relay.

  4. Run Claude Code from your phone. Once paired, your machine shows up as a
  host in the app. Create a new agent, pick provider Claude Code, point it at a
  project folder, and type a task. It executes here on your PC; output streams
  to your phone live. You can also kick one off from the desktop and check in
  from the phone вҖ” same agent, both surfaces.

  Two things worth knowing

  - Run paseo onboard yourself вҖ” I'm deliberately not starting the daemon for
  you, because (a) the QR needs to render in your terminal to be scannable, and
  (b) Paseo's daemon on port 6767 manages live agents, so it shouldn't be
  started/restarted from automation without you driving it.
  - Optional password: since the relay makes your daemon reachable from the
  internet (E2E-encrypted, but still), you can add a second factor with paseo
  daemon set-password. The app will prompt for it on connect. Reasonable to set
  if you're pairing a device you carry around.
